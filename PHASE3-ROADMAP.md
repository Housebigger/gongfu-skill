# 第三阶段规划路线图：从知识仓库到技能共富

日期：2026-06-15
性质：规划文档（meta）。本文档定义第三阶段「技能上架、才华流通、智慧共富」要做什么、怎么做、按什么顺序做。各模块正文经确认后另起目录/文件。

---

## 一、为什么要有这份文档

前两个阶段（思想武装 + 现实建功）已经完成，产出是 1856 个 .md 文件——把伟人思想、国家战略、行业判断、创业方法、协作逻辑、成长路径、机会地图、地域差异，全部沉淀成了普通人能读的文字。

但仔细看 README 第三阶段的表述，它要的不是「再写更多文档」。第三阶段的核心命题是一个**范式跳跃**：

> 把分散在百行百业、沉睡在岗位之间的专业智慧，**蒸馏成**一个个可调用、可验证、可组合、可交易的 `skill`，再把这些 skill **挂牌**到 agent-skillhub 平台之上，让原本锁在岗位里的专业本领，从一次性的体力与脑力消耗，**转化为可持续供给、可被反复调用、可被公平租售的电子服务**。

翻译成一句话：**前两阶段是「人读文档」，第三阶段是「skill 被 AI agent 调用」。**

这个跳跃是否可行，目前是未经验证的假设。这份文档要回答的问题只有一个：**怎么把这个假设变成现实，怎么从 1800 篇文档走向可上架、可流通、可变现的 skill 生态。**

---

## 二、第三阶段与前两阶段的根本区别

| 维度 | 第一/二阶段 | 第三阶段 |
|---|---|---|
| 产出形态 | 文档（人读） | skill（机器调用） |
| 用户门槛 | 要有时间读 + 有能力理解 + 有执行力应用 | 说出需求，agent 调 skill 给判断 |
| 价值交付 | 知识传递（读者自己消化） | 服务交付（skill 直接产出判断/建议） |
| 生产关系 | 作者写 → 读者学 | 专业者蒸馏 skill → agent 调用 → 用户付费使用 |
| 经济模型 | 免费（开源仓库） | 可交易（专业者获回报） |

核心转变：**从「知识仓库」到「技能市场」——从「我读到了这些道理」到「我直接得到了专业判断服务」。**

---

## 三、核心假设与验证路径

第三阶段押注的是一个假设：**这 1800 篇文档里的经验、方法论、行业判断，能被蒸馏成结构化的 skill，被 AI agent 调用后产出有价值的判断。**

验证路径：不要先写规范再做，而是**先做 3—5 个能跑的 skill，从实践反推规范**。

这符合毛泽东方法论——「没有调查就没有发言权」「实践是检验真理的唯一标准」。先做出真东西，再从真东西里提炼标准。先干再总结，用实践定规范。

## 四、五个推进方向

### 方向 1 · skill 设计规范层（基础设施）

**要回答的问题**：一个「好的 skill」长什么样？标准结构是什么？怎么保证可验证、可组合？

**初步内容**：
- skill 标准结构：frontmatter（名称/描述/适用场景/输入输出/依赖）+ 正文（方法论/执行逻辑/边界）+ 验证用例（输入→期望输出）
- 颗粒度原则：一个 skill 解决一个具体问题，不贪大；复杂问题用多个 skill 组合
- 可验证性：每个 skill 必须有「输入→输出」的测试用例，能跑通才算合格
- 可组合性：skill 之间能串联（如「行业扫描 + 创业评估 + 风险校验」）
- 命名/分类/目录约定

**与现有内容的关系**：规范不是凭空设计的，是从方向 2 的实战 skill 制作中**反推**出来的。先做 3—5 个 skill，再做规范。

**方法论根基**：《改造我们的学习》（060）反对教条——规范从实践来，不从理论来。

### 方向 2 · 第一批实战 skill（核心示范）

**要回答的问题**：这 1800 篇文档真的能蒸馏成可调用的 skill 吗？做出来是什么样？

**第一批 skill 候选**（从最成熟的领域提炼，3—5 个起步）：

| skill 名 | 来源 | 输入 | 输出 |
|---|---|---|---|
| `问题诊断` | methodology 矛盾分析/持久战/集中优势兵力 | 你面临的处境/困惑描述 | 主要矛盾判断 + 阶段判断 + 行动方向 |
| `行业机会扫描` | worker_guidance 16 集群 | 你的行业 + 岗位 + 地域 | 增/转/缩信号 + 行动建议 + 风险红线 |
| `创业可行性评估` | entrepreneurship 四条路径 + 劝退 | 你的技能/资源/处境/家庭状况 | 适不适合创业 + 推荐路径 + 阶段标记 + 红线 |
| `协作模式推荐` | collaboration 五种形态 | 你的业务类型 + 合作需求 | 推荐协作形态 + 信任分配要点 + 合规提醒 |
| `成长路线规划` | growth_path 四种画像 | 你的年龄/现状/目标 | 个性化成长地图 + 学习资源 + 关键节点 |

**为什么最优先**：验证核心假设的唯一方法就是做出来。3—5 个能跑的 skill，既证明「文档能蒸馏成 skill」，又自然沉淀出规范。

**方法论根基**：《星星之火，可以燎原》（006）——第一批 skill 就是火种，跑通了就能燎原。

### 方向 3 · agent-skillhub 对接层（技术桥梁）

**要回答的问题**：skill 用什么格式？挂到哪？怎么被调用？

**初步内容**：
- skill 载体格式：SKILL.md 标准 / API 封装 / MCP 工具封装（取决于目标平台）
- 目标平台评估：本地 agent（Hermes/自定义）/ 云端 skillhub / 开放市场
- 调用模式：单 skill 调用 / 多 skill 编排（workflow）
- 测试/调试工具

**与方向 2 的关系**：方向 2 做出的 skill 用什么格式呈现，取决于方向 3 选定的目标平台。两者耦合推进。

**时机**：方向 2 的第一个 skill 做出来后，方向 3 就有了具体对象，而不是空谈。

### 方向 4 · 技能流通与定价层（经济模型）

**要回答的问题**：专业者怎么靠 skill 获得回报？怎么防止白嫖？怎么防止平台垄断？

**初步内容**：
- 定价模式探讨：按次调用 / 订阅 / 按效果 / 免费+增值
- 知识产权：skill 背后的专业经验归谁？怎么保护？
- 收入分配：平台抽成 / 创作者分成 / 开源贡献激励
- 反垄断设计：skill 分发不应被单一平台垄断（呼应仓库反垄断立场）
- 普惠性：skill 不能变成高门槛工具——要让劳动者用得起

**与 README 的对应**：直接对应「可交易」「公平租售」「专业者获得应有回报」「万家灯火共享」。

**时机**：需要等 skill 真做出来、有了实际用户反馈后，才能设计务实可行的经济模型。初期可先探索。

### 方向 5 · 治理与边界层（安全护栏）

**要回答的问题**：怎么防止 skill 被异化？质量怎么治理？红线在哪？

**初步内容**：
- 产品边界（沿用前两阶段红线）：skill 不提供投资建议/交易能力/医疗诊断/法律意见（只提供方向参考）
- 质量治理：低质 skill 不能泛滥；怎么评估/评级/淘汰
- 诚实性要求：skill 的输出必须标注「这是参考不是承诺」
- 防滥用：skill 不被用于欺诈/传销/虚假宣传
- 持续更新：行业在变，skill 怎么持续校准（呼应前两阶段「时局校验」传统）

**方法论根基**：《丢掉幻想，准备斗争》（155）——丢掉「技术天然向善」的幻想，主动建护栏。

## 五、方向间关系

```
    方向 2（第一批实战 skill）
            │
    「先做出来」│「从实践反推」
            ▼
    方向 1（规范层）◄──── 做skill的过程沉淀出规范
            │
            │ 规范指导后续skill
            ▼
    方向 3（对接层）◄──── skill用什么格式取决于目标平台
            │
            │ skill能上架了
            ▼
    方向 4（经济模型）◄──── 有了流通才有经济模型
            │
            │ 贯穿全程
            ▼
    方向 5（治理边界）───── 护栏，从一开始就要有
```

一句话理解：
- **方向 2 是火种**——做出真东西，验证假设
- **方向 1 是标准**——从火种提炼规范
- **方向 3 是管道**——让 skill 能被调用
- **方向 4 是造血**——让专业者获得回报
- **方向 5 是护栏**——防止异化，保证普惠

## 六、实施优先级与顺序

| 顺序 | 方向 | 理由 |
|---|---|---|
| 1 | 方向 2（第一批 skill）+ 方向 5（基本护栏） | 先做真东西 + 同时画红线。护栏不需要等做完才补，从一开始就内嵌 |
| 2 | 方向 1（规范层） | 从第一批 skill 的制作经验中反推规范，不是先写规范 |
| 3 | 方向 3（对接层） | skill 做出来了，选定平台对接 |
| 4 | 方向 4（经济模型） | skill 能上架流通了，再设计务实的经济模型 |

**核心原则：先干再总结。** 第一批 skill 做出来之前，规范/平台/经济模型都是空的。做出来之后，它们才有具体对象。

## 八、诚实的边界

1. 本文档是**规划**，不是已完成的产出。各方向经确认后实施。
2. 第三阶段的核心假设（文档能蒸馏成可调用的 skill）**尚未验证**——第一批 skill 做出来才算验证。
3. 第三阶段不是「再写 1800 篇文档」，是**把已有内容产品化**——从知识仓库变成技能服务。如果做着做着变成了「又写了一堆文档」，说明方向偏了。
4. skill 不是万能的——它放大的是已有经验的质量。1800 篇文档质量好不好，做 skill 的过程会残酷地暴露出来。如果蒸馏不出有价值的 skill，说明前两阶段的内容还需要补强。
5. 经济模型（方向 4）和水之道——既要让专业者获回报，又要防止 skill 变成新的垄断/割韭菜工具。这个平衡是第三阶段最难的部分，也是仓库「共富」立场的试金石。
6. 本规划与 README 第三阶段目标对齐；若第三阶段表述日后调整，本规划相应校准。
