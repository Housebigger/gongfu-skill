# 问题诊断 skill

## 概述

这个 skill 把 methodology/ 的毛泽东战略思维工具箱，蒸馏成一个**可调用的问题诊断服务**。用户描述自己面临的困境或决策，skill 用战略思维工具帮用户看清：主要矛盾是什么、当前处于什么阶段、应该用什么策略。

它解决的问题是：**「我现在面临一个复杂局面，想不清楚该怎么办。」**——不是替你做决定，是给你一套想清楚的工具。

这不是政治学习。它用的是毛泽东著作里那些**经过历史验证的战略思维方法**——矛盾分析、持久战思维、集中优势兵力——帮人在极端复杂和不确定的环境中做出更清醒的判断。

## 什么时候用

- 面临重大决策拿不定主意（转行/创业/搬家/合伙）
- 感觉陷入了困境但说不清问题出在哪
- 多个选择互相矛盾，不知道先抓哪个
- 被情绪裹挟，需要一套冷静的分析框架
- 想验证自己的直觉判断是否靠谱

**不要用于**：具体技术问题（如「这段代码为什么报错」）、即时危机处理（如「现在着火了怎么办」）、专业领域判断（如「该不该买这个基金」）。

## 输入规格

| 参数 | 必填 | 说明 |
|---|---|---|
| 你的困境 | 是 | 用自己的话描述你面临的问题/决策（不用条理清晰，说清楚就行） |
| 你的处境 | 否 | 年龄/行业/家庭/财务等背景 |
| 你纠结的点 | 否 | 你觉得最难决定的那个点是什么 |
| 你倾向的选择 | 否 | 你心里其实偏向哪个方向（说出来不怕错） |

## 执行逻辑

### 第 1 步 · 还原问题

把用户描述的困境，还原成结构化的问题：
- 表面问题是什么？
- 背后真正的矛盾是什么？（往往是用户没说出来的）
- 涉及哪些利益方？（自己/家人/同事/对手）

### 第 2 步 · 矛盾分析（核心）

用《矛盾论》（018）的工具分析：

**主要矛盾是什么？**
- 在用户描述的多个问题中，哪个是「主要矛盾」（解决了它，其他问题会缓解或消失）？
- 哪些是「次要矛盾」（不是不重要，是现在解决不了/不该先碰）？

**矛盾的主要方面是什么？**
- 在主要矛盾中，谁是主导方？（比如「我想转行 vs 家庭需要稳定」——当前主导方可能是家庭需要）

**矛盾会转化吗？**
- 当前的主要矛盾，3 个月后还是主要矛盾吗？（阶段在变，主要矛盾也会变）

### 第 3 步 · 阶段判断

用《论持久战》（027）的工具判断当前阶段：

| 阶段 | 特征 | 策略 |
|---|---|---|
| 速决战阶段 | 机会窗口很短，必须快速行动 | 果断出手，不犹豫 |
| 相持阶段 | 最难熬的阶段，看不到进展但不能退 | 蓄力等待，不冒进不放弃 |
| 反攻阶段 | 形势开始好转，该乘势而上 | 集中力量，扩大战果 |

**大多数人的困境是因为误判了阶段**——在相持阶段用了速决战策略（焦虑、急躁），或在速决战阶段用了持久战策略（犹豫、等待）。

### 第 4 步 · 策略推荐

根据矛盾分析和阶段判断，推荐策略工具：

| 你面临的情况 | 用哪个工具 | 核心原则 |
|---|---|---|
| 方向太多不知选哪个 | 《集中优势兵力》（103） | 聚焦 1—2 个，不撒胡椒面 |
| 想做但怕失败 | 《一切反动派都是纸老虎》（229） | 战略上藐视困难，战术上重视准备 |
| 该不该依赖某人/某平台 | 《丢掉幻想》（155） | 丢掉「永远可靠」的幻想，准备后路 |
| 长期看不到效果想放弃 | 《论持久战》（027） | 这是相持阶段，硬仗是持久战 |
| 不确定是不是真需求 | 《关心群众生活》（011） | 看人是不是真的在为这个事发愁 |
| 想赌一个未来方向 | 《星星之火》（006） | 看火种保不保得住、方向对不对 |

### 第 5 步 · 幻想排查

检查用户是否存在需要丢掉的幻想：
- 「换个环境就能解决一切」→ 问题可能在自己不在环境
- 「再等等就会好」→ 如果趋势在恶化，等的代价越来越大
- 「我什么都能做」→ 集中优势兵力，什么都想做等于什么都没做
- 「别人都这样」→ 别人的处境和你的不同

## 输出规格

```
## 问题诊断结果

### 你的问题还原
（用一句话说清楚你真正的问题是什么）

### 主要矛盾
（你当前的核心矛盾是什么——不是你列出的所有问题，是那个最关键的）

### 阶段判断
（你当前处于速决战/相持/反攻的哪个阶段）

### 策略工具推荐
（推荐 1—2 个最适合的战略思维工具 + 怎么用）

### 需要丢掉的幻想
（如果有，明确指出）

### 行动方向
（不是具体答案，是「往哪个方向想/往哪个方向走」的指引）

### 一句话总结
（用毛泽东方法论的一句话收尾）
```

## 测试用例

### 用例 1：30岁程序员想转行但怕

输入：困境=「我做了8年Web开发，感觉被AI威胁想转行，但又怕转错。同时家里有房贷，不能断收入。」

期望诊断：
- 还原：表面是「转不转行」，真正的主要矛盾是「技能贬值风险 vs 即期收入需求」
- 阶段：相持阶段——不能裸辞（速决战），也不能不动（亡国论）
- 工具：《论持久战》——边上班边学新方向，不赌速胜
- 幻想排查：丢掉「Web开发永远安全」的幻想
- 方向：不转行而是「升行」——从纯写代码转「行业+AI」复合（growth_path 线二/三）

### 用例 2：45岁传统零售想创业

输入：困境=「我开了15年小超市，生意越来越差，想关店做电商，但不会搞。家人说我瞎折腾。」

期望诊断：
- 还原：表面是「开店还是做电商」，主要矛盾是「传统零售模式失效 vs 新能力还没建立」
- 阶段：速决战→不是，是相持转反攻的前夜。旧模式在死，新模式还没立起来
- 工具：《矛盾论》抓主要矛盾——不是学电商技术，是搞清楚「我的客户去哪了」
- 幻想排查：丢掉「开了15年就不会倒」的幻想
- 方向：先用 entrepreneurship 路径二（内容化）或路径三（电商化）小成本试，不关店先加副线

### 用例 3：应届生选择太多

输入：困境=「我毕业了，考公、进企业、考研、回老家、出国都想过，什么都想抓，什么都定不下来。」

期望诊断：
- 还原：表面是选择太多，主要矛盾是「缺乏判断自身禀赋与市场需求匹配度的方法」
- 工具：《集中优势兵力》——选一个确定性最高的方向扎进去，不撒胡椒面
- 幻想排查：「什么都想抓」本身就是幻想——集中优势兵力一次只打一个点
- 方向：先用 industry-scan skill 扫一遍行业，再用 growth_path 线一（应届生）的确定性第一原则

## 诚实的边界

1. 本 skill 提供的是**思维工具**，不是答案。它帮你看清主要矛盾和阶段，但最终决策仍由你做。
2. 方法论工具来自毛泽东著作的**战略思维方法**（矛盾分析/持久战/集中优势兵力），不是政治教条。
3. 本 skill 不替代心理咨询/法律咨询/专业职业规划。
4. 如果用户的问题涉及心理健康危机，应建议寻求专业帮助而非用本 skill。
5. 「丢掉幻想」不等于「打击信心」——目的是让人清醒，不是让人绝望。
