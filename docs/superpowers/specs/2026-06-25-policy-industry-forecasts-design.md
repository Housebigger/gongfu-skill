# 设计文档：经济政策专栏 → 逐集群行业前景推演（知识层·首批 6 集群）

- 日期：2026-06-25
- 状态：已通过设计评审，待落地
- 涉及层：`strategy/references/policy_archive/`（原料库扩充）+ `strategy/economic_policy/`（分析池新层）。**纯知识内容，不动引擎、不升版本。**

## 1. 背景与目标

经济政策专栏（战略库第三根源）已完成 Phase 1（台账 01 + 主线复盘 02/03 + 作用规律 04 + 边界 05）与 Phase 2 知识层（推演方法 06 + 宏观议题推演 07/08）+ Phase 2 引擎接入（v1.4.0，注入 06 方法框架）。当前 07/08 的推演停在**宏观议题层**（内需/房地产/财政地方债/新动能/外需出海/人口劳动力），尚未落到**具体行业**。

**目标**：扩充原料库 + 分析池，新增一层"**政策机制驱动·逐产业集群·三情景前景推演**"——把 07/08 的宏观议题推演**下沉**到具体产业集群，为更多行业形成可落地的前景判断与劳动者视角翻译。

## 2. 已确认的设计决策

- **组织轴 = 按引擎 16 产业集群（A–P）**（用户已选）：每集群一份"政策驱动三情景前景推演"，对齐 cluster IDs，为日后按 cluster 接入引擎留接口。
- **本轮覆盖 = 6 个高政策敏感集群**（用户已选）：C 绿色能源全链 / K 传统重化工与建材 / E 民生服务 / G 基建物流房地产 / A 先进制造与硬科技 / D 农业与乡村振兴。其余 10 个分批。
- **本轮交付 = 仅知识层**（用户已选）：引擎接入另开 spec，延续"先内容后引擎"节奏。
- **原料来源 = 先搜后确认**（用户已选）：实现时各集群在线核实行业政策与官方数据，带来源；抓不到的只登记不伪造。
- **方法复用**：每集群推演复用既有纪律——`04` 规律引擎 + `06` 六要素法 + `05/06` 诚实边界，零新方法。

### 5.1 每个集群文件的统一六要素（镜像 06 §1）

1. **现状锚点**：该集群当前行业事实 + 锚点政策 + 官方数据（带口径与来源，先搜后确认）。
2. **规律库外推**：明确引用 `04` 的哪条规律（时滞分层 / 传导节点 / 就业收入传导）外推本集群。
3. **套外部约束 + 国情**：套 `06` §2/§3 的国际/国情共用假设，对本集群做顺风/逆风/待观察的方向定性。
4. **基准 / 上行 / 下行三情景**：各带触发条件（可观察指标）+ 关键假设 + 对劳动者方向含义；一律"若…则…"条件式，不给时间表点位。
5. **可证伪点 / 观察指标**：2–4 个官方可查指标，说明看到什么数据说明在走哪条情景。
6. **劳动者含义**：落到岗位/技能/区域/卡位窗口；挂 `industry-signals.yaml` 该集群的 growth_roles/shrink_roles + 产业链卡点（Serenity）思路；含"下行信号出现时留意什么"。

## 6. 诚实纪律（本层的魂）

- 情景非预言、给方向不给时间表、不投资建议/不出个股、不做政治预测（承接 `05` + `06` §4）；
- 政策↔结果是**观察性关联非严格因果**（"伴随""在…之后观察到"，不用"由于…导致"）；
- 数据限国家统计局/央行/财政部/各行业主管部门官方渠道，带口径与时间窗口；抓不到的只登记、标"待核实"，不伪造；
- **中立**：只记录政策原文与可查结果，不用"伟大/英明/必将"，也不作影射式批评；
- **劳动者视角翻译**（宏观→岗位/技能/区域，承接 `00` 第三节）是每篇落点，不写给决策者/投资者看的分析。
