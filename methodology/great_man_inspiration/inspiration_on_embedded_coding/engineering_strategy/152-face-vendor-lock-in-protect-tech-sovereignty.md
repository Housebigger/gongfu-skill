# 从《中国人民解放军总部发言人为英国军舰暴行发表的声明》看嵌入式开发：面对外部约束——守住你的技术主权

- 日期：2026-06-13
- 原文对应：《中国人民解放军总部发言人为英国军舰暴行发表的声明》（1949 年 4 月 30 日）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19490330.htm
- 写作背景：英舰闯入长江开炮，毛泽东据理力争，不接受任何威胁。新中国的第一份外交声明。
- 说明：本文将"面对威胁守住主权"映射到嵌入式开发中面对供应商/客户不合理技术约束时的应对。
- 核心思想当代转译：供应商说"必须用我们的方案"？客户说"不能改动硬件"？先评估合理性，不合理的据理力争。

---

## 一、面对供应商的技术绑定

- "必须用我们的 SDK"→ 评估是否有替代方案，有就用开放的
- "不支持你们的平台"→ 要求提供 API 而非 SDK
- 关键：不被单一供应商绑架

## 二、守住技术选型主权

- 不因时间压力而接受不合理的技术约束
- 不因供应商的声量而放弃更好的方案
- 技术选型权 = 项目主权

## 三、给今天每个被供应商技术绑定的工程师

你的技术选型权就是你的"长江"。可以合作，不能被绑架。
