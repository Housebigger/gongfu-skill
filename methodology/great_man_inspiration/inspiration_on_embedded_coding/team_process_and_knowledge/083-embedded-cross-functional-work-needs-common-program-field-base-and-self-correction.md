# 从《论联合政府》看嵌入式团队：跨团队攻坚要有共同纲领、现场基础和自我纠错

日期：2026-05-13

原文对应：
《论联合政府》
原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19450424.htm

写作背景：
1945年4月24日，毛泽东在中国共产党第七次全国代表大会上作政治报告《论联合政府》。这篇文章承接前一天《两个中国之命运》的判断，进一步回答中国应该怎样走向胜利和建国：成立民主的临时联合政府，实行民主改革，动员和统一全国抗日力量，打败日本侵略者；在胜利后建设独立、自由、民主、统一和富强的新中国。文章系统总结抗日战争中的两条路线，提出新民主主义的一般纲领和具体纲领，并在结尾强调理论联系实际、密切联系群众、全心全意为人民服务、批评和自我批评，以及在民主集中制下团结起来完成任务。

说明：
转译到嵌入式工程，这篇文章适合用在新平台、新整机、新量产线、客户现场攻坚、硬件固件测试产线多方协作的阶段。嵌入式项目最怕各团队都说支持，但没有共同纲领：硬件按硬件节奏，固件按固件想象，测试最后补，产线临时救，FAE现场兜底，客户需求被二手转述。083提醒我们，真正能完成大任务的组织，必须把各方力量联合到一个明确纲领和纪律之下，并保持来自现场的反馈与自我批评。

核心思想的当代转译：
如果把《论联合政府》翻成嵌入式工程语言，我会概括成一句话：嵌入式项目进入平台化、量产或现场攻坚阶段，不能靠单一团队英雄主义；要建立硬件、固件、测试、产线、供应链、FAE和客户现场共同认可的工程纲领，用广泛协作、现场干部、民主集中和自我批评，把系统交付做成真实秩序。

## 一、项目窗口出现，不等于工程系统已经统一

083开头说形势有利，但中国仍然不团结。嵌入式项目也常这样：新板回来、主芯片到货、客户样机通过、量产窗口临近，但工程系统并不统一。

硬件可能追求快速改版，固件可能追求功能闭环，测试可能还没有边界用例，产线可能没有治具，供应链可能没有替代料，现场可能没有日志工具。机会窗口越近，越需要共同纲领。

## 二、共同纲领要落到接口、测试、产测和现场规则

工程纲领不能只写“保证质量”“按期量产”“客户满意”。它要具体到：硬件接口冻结规则，固件版本节奏，测试覆盖范围，产测治具和脚本，关键器件替代策略，日志和故障码标准，OTA回退方案，现场问题分级，客户需求变更入口。

这些条款就是嵌入式项目的“共同纲领”。没有它，多团队协作只会变成互相等待和互相甩锅。

## 三、联合真实工程力量，而不是只让负责人开会

083的联合，是把真实抗日和建国力量组织起来。嵌入式项目里的真实力量包括硬件工程师、固件工程师、测试、产线技术员、采购、供应商、FAE、维修、客户现场人员。

如果会议只在项目经理和几位负责人之间循环，一线异常、批量差异、现场环境、供应商限制就进不了系统。真正的联合，是让这些事实有入口、有负责人、有闭环。

## 四、民主集中制对应工程里的充分暴露和统一执行

嵌入式项目需要先让风险充分暴露：电源余量、温漂、EMC、存储寿命、网络异常、产线误操作、现场升级失败，都要能被说出来。然后再形成统一的版本节奏和冻结纪律。

只集中不民主，会压住风险；只民主不集中，会版本失控。工程组织要做到：讨论时人人拿证据，决策后按同一版本和同一标准执行，复盘时允许事实推翻旧方案。

## 五、培养本地干部，就是培养现场可解决问题的人

083强调从本地人民中培养本地干部。嵌入式项目也要培养现场干部：产线有人能判断常见失败，FAE有人能按手册定位，维修有人能反馈返修规律，客户现场有人能采集关键日志，区域支持有人能处理升级风险。

如果所有问题都回到总部核心工程师，系统就没有真正长大。量产和现场稳定，靠的是本地问题本地初步闭环，再把规律反馈到平台。

## 六、自我批评防止工程组织被“能跑就行”腐蚀

083说工作和思想都要经常打扫。嵌入式团队的灰尘，就是样机跑通后忽视边界、客户演示成功后轻视产测、一次现场救火成功后不补工具、负责人怕丢脸不承认设计误判、团队把测试反馈当找茬。

每个阶段都要问：我们有没有用样机成功替代量产证据？有没有让现场替总部背锅？有没有把供应链风险藏起来？有没有用经验主义压过数据？有没有把复盘做成免责材料？

083给嵌入式团队的提醒是：跨团队攻坚不是靠某个英雄工程师顶住，而是靠共同纲领、现场基础、统一纪律和持续自我纠错，把复杂系统真正组织起来。
