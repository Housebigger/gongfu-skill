# inspiration_on_software_development

这个主题把毛主席相关文章转译成面向现代软件开发的方法论，重点服务于 App、Web、全栈、微信小程序和一般产品工程场景。

## 主题边界

这里处理：
- 产品战略与需求判断
- 用户研究与需求澄清
- 架构与工程能力建设
- 迭代节奏、交付组织与项目主战场判断
- 团队流程、协作治理与质量机制

不处理：
- 固件、驱动、真机 bring-up、板级调试等硬件强耦合问题

那些内容属于 `../inspiration_on_embedded_coding/`。

## 子目录速览

### `product_strategy/`
适合看：
- 产品楔子、主战役、主矛盾与战略扩张

### `user_research_and_requirements/`
适合看：
- 用户分层
- 调研先行
- 研究—构建—验证闭环
- 如何避免围着二手汇报和 dashboard 对产品问题悬空定性

### `architecture_and_engineering/`
适合看：
- 核心能力建设
- 系统分析与路线图

### `iteration_and_delivery/`
适合看：
- 资源供给线
- 主战场组织
- 项目阶段判断
- 长期承压下的交付推进

### `team_process_and_quality/`
适合看：
- 团队协作治理
- 批评反馈机制
- 跨部门支持面经营
- 保护建设者、抑制软退与防止伪协同
- 如何把团队学习从背框架改成研究现状、研究历史、把理论变成会用的方法

## 使用提醒

- 当前这个主题已不再是“预留目录”，而是仓库里体量最大的成熟主题之一
- 如果一个主题主要服务于用户型软件产品，而不是嵌入式硬件现场，就优先放这里
