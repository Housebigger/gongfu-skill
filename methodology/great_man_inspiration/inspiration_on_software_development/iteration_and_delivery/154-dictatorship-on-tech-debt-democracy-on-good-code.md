# 从《论人民民主专政》看软件开发：对技术债专政、对好代码民主

- 日期：2026-06-13
- 原文对应：《论人民民主专政》（1949 年 6 月 30 日）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19490630.htm
- 写作背景：建国前夕提出"对人民民主、对敌人专政"的治国原则。
- 说明：映射到技术管理——对技术债和坏代码"专政"，对好实践和贡献者"民主"。
- 核心思想当代转译：技术债是敌人，果断清掉；好代码和贡献者是人民，充分奖励。

---

## 一、对技术债"专政"

- 识别→ 每次 code review 标记
- 量化→ 记录技术债列表和影响
- 清除→ 每个 sprint 固定 20% 时间还债

## 二、对好实践"民主"

- 鼓励→ 公开表扬好的 PR
- 推广→ 分享优秀实践到团队
- 保护→ 给写好代码的人更多自主权

## 三、给今天每个被技术债困扰的团队

把技术债当敌人，把好实践当人民。对敌人专政，对人民民主。
