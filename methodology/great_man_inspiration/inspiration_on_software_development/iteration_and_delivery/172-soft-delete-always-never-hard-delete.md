# 从《镇压反革命必须实行党的群众路线》看软件开发：宁可漏报不可误删——数据操作的底线

- 写作背景：可捕可不捕的不捕。
- 说明：映射到数据操作底线。
- 核心思想当代转译：删除数据、封禁用户、下线服务——可删可不删的不删。宁可多存一条数据，不可误删一条。

## 一、核心要点

- 软删除→ 永远不要 hard delete
- 可封可不封的不封→ 先观察再行动
- 严格审查→ 每次删除操作都要审批

## 二、给今天每个想"清理一下数据库"的开发者

别清理。硬盘不值钱，数据无价。
