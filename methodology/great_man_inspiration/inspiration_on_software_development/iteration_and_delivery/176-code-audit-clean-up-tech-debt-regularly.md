# 从《关于"三反"、"五反"的斗争》看软件开发：代码审计——定期清理技术债

- 日期：2026-06-13
- 原文对应：《关于"三反"、"五反"的斗争》（1951 年 11 月—1952 年 3 月）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-195111.htm
- 写作背景：反贪污、反浪费、反官僚。分类处理。
- 说明：映射到代码审计清理技术债。
- 核心思想当代转译：你的代码库里有没有"贪污"（抄袭代码）、"浪费"（无用依赖）、"官僚主义"（过度设计）？定期代码审计，分类处理：小问题记TODO，大问题立即重构。

---

## 一、核心要点

- 反"贪污"→ 别copy-paste，写可复用组件
- 反"浪费"→ 删无用依赖
- 反"官僚"→ 别过度设计
- 分类处理→ 小问题记录，大问题立即修

## 二、给今天每个"先这样吧以后再说"的团队

以后=永远不会。定期审计，立即处理。
