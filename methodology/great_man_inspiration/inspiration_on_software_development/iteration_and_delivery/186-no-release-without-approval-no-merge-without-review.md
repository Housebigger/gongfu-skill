# 从《破坏纪律的批评》看软件开发：守纪律——不经审批别发布，不经review别合并

- 日期：2026-06-13
- 原文对应：《对刘少奇、杨尚昆破坏纪律擅自以中央名义发出文件的批评》（1953 年 5 月 19 日）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19530519.htm
- 说明：映射到代码审批纪律。
- 核心思想当代转译：不经审批就发布=生产事故。不经review就合并=代码质量失控。守纪律=所有变更经过审批。

---

## 一、核心要点

- 经审批才能发布→ 别跳过流程
- 不经review=无效→ code review不是可选项
- 破坏纪律=破坏信任→ 团队信任一旦破裂很难修复
- 检查过去→ 回头看看有没有跳过流程的代码

## 二、给今天每个"紧急hotfix不需要review"的团队

紧急hotfix更需要review。因为紧急=更容易犯错。
