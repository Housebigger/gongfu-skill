# 从《关于《红楼梦》研究问题的信》看软件开发：新人能挑战权威代码——别被"祖传代码"绑架

- 日期：2026-06-13
- 原文对应：《关于《红楼梦》研究问题的信》（1954 年 10 月 16 日）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19541016.htm
- 说明：映射到技术权威挑战。
- 核心思想当代转译：两个青年团员挑战权威学者——"这是三十多年以来向所谓权威的第一次认真的开火。"代码库也一样——"祖传代码"往往是"权威"，没人敢质疑。但新人往往能看出问题。"大人物"（老员工）往往拦阻——"这代码没问题，别动"。

---

## 一、核心要点

- 新人能看出老手看不到的问题→ 新视角有价值
- 权威代码也要质疑→ 老≠对
- "大人物"会拦阻→ 既得利益者抵制变化
- 别当旧代码的"俘虏"→ 不被"一直是这样的"绑架

## 二、给今天每个"这段代码不能动因为XXX写的"的团队

代码的好坏不看谁写的，看它本身质量。新人质疑值得认真对待。
