# 从《永远保持艰苦奋斗的作风》看软件开发：上线不是终点——永远保持工程纪律

- 日期：2026-06-13
- 原文对应：《永远保持艰苦奋斗的作风》（1949 年 10 月 26 日）
- 原文链接：https://www.marxists.org/chinese/maozedong/marxist.org-chinese-mao-19491026.htm
- 写作背景：永远保持艰苦奋斗。
- 说明：映射到产品上线后不放松工程纪律。
- 核心思想当代转译：上线是最容易的时候。运维、监控、持续迭代才是真正的艰苦奋斗。

---

## 一、上线后的松懈

- 不写测试了→ "线上跑着就行"
- 不做 code review→ "赶进度"
- 不做监控→ "出问题了再说"

## 二、给今天每个在上线后放松的团队

上线是开始，不是结束。
