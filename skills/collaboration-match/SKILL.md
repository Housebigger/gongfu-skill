# 协作模式推荐 skill

## 概述

这个 skill 把 strategy/collaboration/ 的五种协作形态 + 信任分配逻辑，蒸馏成一个可调用的协作建议服务。用户描述自己的业务和合作需求，skill 推荐最适合的协作方式、分钱规则和合规底线。

它解决的问题是：**「我一个人做不了了，该找什么样的合作？怎么合作不散伙？怎么分钱不打架？」**

## 什么时候用

- 业务增长遇到个人时间/能力天花板
- 想接更大的单但一个人吃不下
- 考虑找人合伙但不知道怎么分
- 已经在合作但出了矛盾想理清
- 想从单干升级为团队/组织

**不要用于**：公司法务咨询、股权结构设计（需要专业律师）、劳动纠纷处理。

## 输入规格

| 参数 | 必填 | 说明 | 示例 |
|---|---|---|---|
| 业务现状 | 是 | 你现在做什么、月收入大概多少 | 上门维修月入8000 |
| 合作需求 | 是 | 你缺什么/想要什么合作 | 缺会做内容的人/想接更大的单 |
| 现有伙伴 | 否 | 已经有合作的人吗 | 有个同行聊过 |
| 你的核心能力 | 否 | 你最擅长什么 | 技术/获客/运营 |
| 风险偏好 | 否 | 保守/中等/激进 | 保守不想投钱 |

## 执行逻辑

### 第 1 步 · 天花板诊断

判断用户是否真的到了需要协作的阶段：

| 信号 | 意味着 |
|---|---|
| 开始拒单（订单多到接不过来） | ✅ 该协作了 |
| 客户需要完整方案但只有部分能力 | ✅ 该协作了 |
| 反复卡在同一个能力瓶颈 | ✅ 该协作了 |
| 看到一个人吃不下的大机会 | ✅ 该协作了 |
| 只是觉得累/孤单 | ❌ 还不到时候，先优化个人效率 |
| 自己还没跑通 | ❌ 先单干跑通再找人 |

**如果未到协作阶段，明确建议先做个人优化，不硬推协作。**

### 第 2 步 · 形态匹配

根据需求匹配五种协作形态：

| 形态 | 松紧度 | 匹配条件 |
|---|---|---|
| 技能互换 | 最松 | 互相补短板，不想涉及钱 |
| 互助小组 | 松 | 同行抱团，共享信息/拼单/互推客户 |
| 接包小队 | 中 | 有大单/复杂项目，临时组队 |
| 合伙经营 | 紧 | 业务稳定，要长期投入 |
| 合作社 | 最紧 | 农业规模化/社区服务 |

**铁律**：从松到紧逐步升级，不跳级。第一次合作从技能互换或互助小组开始。

### 第 3 步 · 分配建议

如果涉及分钱（接包小队以上），给出分配框架：

**合作第一天必须谈的七件事**（来自 collaboration/03）：
1. 各自出什么（钱/力/资源/技术）
2. 谁拍板（决策权——忌讳50/50，必须有一个人占51%+）
3. 怎么分钱（比例+周期+留存）
4. 怎么算账（谁管账+多久对一次+随时可查）
5. 谁干什么（分工不重叠不真空）
6. 亏了怎么办（止损线）
7. 谁想走怎么办（退出机制）

### 第 4 步 · 红线提醒

无论什么协作形态，必须输出三条红线：
1. 不向公众集资（非法集资红线）
2. 不搞多级拉人头（传销红线）
3. 雇人就签合同交社保（非法用工红线）

## 测试用例

### 用例 1：维修工想接大单

输入：业务=上门家电维修月入8000，需求=有客户要全屋翻新但我只会水电

期望推荐：
- 天花板诊断：✅（客户需要完整方案但只有部分能力）
- 形态：接包小队（临时组队接这一单）
- 建议：你当总包（水电是你），找瓦工+木工组队
- 分钱：提前定好各工种分账比例，你是总包对客户负责

### 用例 2：内容创作者想合伙

输入：业务=做农技短视频月入5000，需求=想开农资店但不会选品运营

期望推荐：
- 天花板诊断：✅（看到一个人吃不下的大机会）
- 形态：合伙经营（长期合作）
- 建议：从互助小组起步（先合作试水3-6月），再升级
- 红线：先小人后君子，七件事写进备忘录

### 用例 3：还没跑通就想找人

输入：业务=刚辞职想做电商还没开始，需求=想找人一起做

期望推荐：
- 天花板诊断：❌（自己还没跑通）
- 建议：先一个人跑通最小可行模式，跑通了再找人
- 理由：拉人入伙时你没有一个被验证的模式，浪费别人时间也消耗你信用

## 诚实的边界

1. 本 skill 给的是**协作方向**，不是法律文件。涉及注册/协议/税务务必咨询专业人士。
2. **协作比单干复杂**——如果连单干都没跑通，不要急着合伙。
3. 本 skill 不替代公司法务/股权设计/劳动法咨询。
4. 分配建议是框架参考，具体比例需合作各方自行协商。
