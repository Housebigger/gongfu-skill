# 成长路线规划 skill

## 概述

这个 skill 把 strategy/growth_path/ 的四种起点画像 + 学习地图三层结构，蒸馏成一个可调用的个人成长规划服务。用户输入自己的现状和目标，skill 产出一条个性化的成长路线。

它解决的问题是：**「我这个人，站在现在这个位置，想走到那里，该怎么一步步走？」**

## 什么时候用

- 不知道该学什么技能
- 有目标但不知道怎么一步步达到
- 想考证但不知道考哪个、先后顺序
- 面临人生节点（毕业/30岁/转行/返乡）想规划
- 想知道当前阶段该聚焦什么

**不要用于**：具体培训机构的推荐（skill 只给方向不给商业推荐）、学历教育规划。

## 输入规格

| 参数 | 必填 | 说明 | 示例 |
|---|---|---|---|
| 现状 | 是 | 年龄+当前岗位/身份+行业 | 35岁产线操作工/22岁应届生 |
| 目标 | 否 | 想达到什么状态 | 升技术岗/转行/创业/回县城 |
| 地域 | 否 | 影响资源推荐 | 深圳/中部县城/东北 |
| 已有技能 | 否 | 已经会什么 | 会基础电工/什么都不会 |
| 时间预算 | 否 | 每天能投入多少学习时间 | 上班族每天1-2小时 |

## 执行逻辑

### 第 1 步 · 画像匹配

把用户匹配到四种画像之一或组合：

| 画像 | 匹配条件 |
|---|---|
| 应届毕业生 | 20—25岁，刚毕业或即将毕业 |
| 在岗产线工人 | 28—38岁，制造业一线，有5—10年经验 |
| 跨行转行者 | 38—50岁，原行业下行，需要转行 |
| 县域/返乡劳动者 | 任何年龄，在县城或想返乡 |

### 第 2 步 · 三层结构定位

判断用户在学习地图三层结构的哪一层：

```
第三层 · 判断力/经营能力（长期积累）
        ↑ 你在这里？
第二层 · 证书/资质（6个月—2年）
        ↑ 还是在这里？
第一层 · 手艺（1—3年）← 大多数起步者在这里
```

**规则**：如果用户没有第一层（手艺），优先补第一层。不要跳级。

### 第 3 步 · 路线生成

根据画像+目标，生成一条分阶段的路线。每阶段包含：
- 学什么（具体技能）
- 考什么证（如有）
- 多久见效
- 阶段目标（可验证的里程碑）
- 免费/低成本学习资源

### 第 4 步 · 关键节点标注

标出路线上的关键决策点：
- 什么时候该考证
- 什么时候该考虑转管理
- 什么时候该考虑独立/创业
- 什么时候该止损换方向

### 第 5 步 · 持久战预期管理

明确告诉用户：成长是持久战，不是速成。给出现实的时间预期（学一门手艺至少1—3年），反对速胜论。

## 输出规格

```
## 成长路线规划结果

### 你的画像
画像类型：___  当前层级：第___层

### 你的成长路线
第一阶段（___个月）：学___ → 目标___
第二阶段（___个月）：学___ → 目标___
第三阶段（___）：___

### 关键节点
- 节点1：什么时候做___
- 节点2：什么时候考虑___

### 学习资源
（免费/低成本资源清单）

### 现实预期
（持久战时间线——学成至少需要多久）

### 一句话总结
```

## 测试用例

### 用例 1：35岁产线工人想升级

输入：现状=35岁产线操作工干了10年，目标=升技术岗，地域=长三角

期望路线：
- 第一阶段（2个月）：考低压电工证（入场券）
- 第二阶段（6—12月）：学PLC基础→设备维护实操
- 第三阶段（1—2年）：成为独当一面的维修工程师
- 节点：拿到电工证后可内部申请转维护岗
- 预期：2—3年完成质变

### 用例 2：22岁应届生选方向

输入：现状=应届毕业生不知道做什么，目标=选一个确定方向

期望路线：
- 先用industry-scan确定方向
- 确定性优先：硬科技/刚性服务/增量赛道
- 第一阶段：选准方向后扎进去3—5年
- 预期：25岁前最大的风险是什么都不选

### 用例 3：什么都不会的待业者

输入：现状=28岁待业什么都不会，目标=找一条出路

期望路线：
- 先补第一层（手艺）——不能跳级
- 推荐低门槛+刚性需求的手艺（电工/护理/烹饪）
- 第一阶段（1—3月）：培训+考证
- 预期：先有饭碗再谈发展

## 诚实的边界

1. 本 skill 给的是**方向+节奏**，不是精确到天的计划。
2. 成长是持久战——任何承诺「速成」的方向都不在本 skill 推荐范围。
3. 学习资源信息基于 2025—2026 年，平台/政策会变。
4. 本 skill 不替代专业学历教育规划。
