# strategy/ — 战略库（第二阶段：现实建功）

> 全民共享，共同富裕。

战略库是共富参谋的第二个底层思想源。如果说 `methodology/`（方法论库）解决"用什么思想武器看世界、做判断"，那么 `strategy/` 解决"在新时代中国的发展主战场上，各行各业的劳动者具体该往哪走、怎么干"。它紧扣"十五五"规划的发展方向、产业任务与时代机遇，把宏观规划逐层翻译成普通人可理解、可执行、可落地的现实指引。战略库现有**三条核心研究根源**：十五五规划（国家宏观方向）+ Serenity 产业链分析法（产业链微观卡点）+ 经济政策追踪专栏（政策↔现实作用规律）。

## 它在整个仓库里的位置

```
methodology/ + strategy/   ← 两个底层思想源（人来读、人来提炼）
        │
        │  人工蒸馏（无脚本，手写进 YAML）
        ▼
   skills/data/*.yaml       ← 引擎真正读取的结构化知识
        │
        ▼
   引擎 engine/ → Hermes 插件 / MCP / HTTP API / 静态知识包
```

**重要**：`strategy/` 的内容在引擎运行时**不被直接读取**——它是源料，经人工提炼进 `skills/data/*.yaml` 后才进入引擎。各 `SKILL.md` 顶部的 `source_layers:` 字段标注的就是"该 skill 的判断蒸馏自哪些 strategy 子层"，用于追溯，不是代码引用路径。

## 与方法论库的分工

- `methodology/`（思想武器）回答：**怎么想**——矛盾分析、实事求是、剩余价值、新质生产力……
- `strategy/`（现实指引）回答：**往哪走、怎么干**——哪个产业在增、哪条创业路径适合你、哪种协作怎么分钱、哪个区域有什么机会。

两者在 `skills/data/` 层汇合：方法论提供判断工具，战略库提供行业/区域/路径事实，共同支撑 7 个 skill 的输出。

## 维护原则

- 每个子层以自己的 `README.md` 为导航入口，本文件只做总览
- 编号约定保持稳定：`00` 恒为方法论框架，其后顺序展开
- 内容按"用途/母题"放置，不按日期
- README 只描述真实存在的文件
