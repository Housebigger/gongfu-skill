## 一、为什么要写这份文档

`README.md` 把仓库宗旨的第一句话定为——**「促进广大劳动人民具备零成本创业的意愿和条件」**。「零成本创业」是整个仓库的灵魂词。

但回看 strategy 层现状，沉淀下来的几乎全是**「在岗/转行」视角**：规划解读（国家往哪走）+ 行业行动指南（某行业的一线劳动者怎么办）。这两层解决的是「看清趋势」和「站稳脚跟」，是打工人和转行者的视角。

而 `README.md` 第二阶段那句其实是一条完整的动作链：

> 看清趋势 → 站稳脚跟 → 顺势成长 → 借势创业 → 结成协作 → 开辟新业 → 做出新蛋糕

这条链子的后半段（成长 / 创业 / 协作 / 新业 / 新蛋糕）目前要么零散、要么空白。尤其是「零成本创业」这个灵魂词，在 strategy 层没有专门的模块对应。

这份文档要回答的问题只有一个：**为了让 strategy 层配得上仓库宗旨和第二阶段动作链，还应该新增哪些层，各自负责什么，彼此什么关系，按什么顺序做。**

## 三、动作链缺口定位

把第二阶段动作链逐个动词对到现状：

| 动作链动词 | 现状覆盖 | 缺口判断 |
|---|---|---|
| 看清趋势 | ✅ analysis + worker_guidance 行业总判断 | 无 |
| 站稳脚跟 | ✅ worker_guidance 转型路线图 / 风险红线 | 无 |
| 顺势成长 | ◐ 零散在岗成长建议 | 缺「个人成长地图」系统化 |
| 借势创业 | ❌ 仅个别集群「管理/创业岗」层带了一句 | **最大空白**，且正对灵魂词 |
| 结成协作 | ❌ 完全空白 | 个人视角无法覆盖集体行动 |
| 开辟新业 | ◐ 增量赛道提了，未讲「新业态怎么从零孵化」 | 与创业层耦合 |
| 做出新蛋糕 | ❌ 完全空白 | 非零和创造导向缺位 |

结论：链子后半段是「创造与协作」导向，现状是「择业与转型」导向。**视角上的系统性错位**，是当前 strategy 层最大的结构性缺口，也是本规划要补的目标。

## 四、新增层规划（五个方向）

> 命名沿用项目既有约定：英文目录名 + `00-方法论框架.md` 开篇 + 各专题文件，每层自带 README 作总索引。

### 层 1 · entrepreneurship（零成本创业落地层）

- **核心问题**：一个没有资本、没有团队、只有一身技能和劳动经验的普通劳动者，怎么在 2026—2030 年这个窗口里，用接近零成本的方式启动一份自己的小事业。
- **为什么最优先**：直接补上「零成本创业」这个灵魂词在 strategy 层的空白；且能复用 worker_guidance 已有的 16 集群【增】信号，不是从零开始。
- **与现有层的关系**：worker_guidance 回答「在某行业打工怎么办」，entrepreneurship 回答「怎么从打工人/零起步做成自己的事」。前者是行业切片，后者是行动模式切片，交叉点（某行业的创业机会窗口）互相引用，不重复。
- **初步内容结构**：
  - 通用方法论：零成本/极低成本启动的现实路径（服务化、内容化、电商化、技能出租）
  - AI 工具如何把内容/服务/电商的启动成本压到接近零
  - 各集群创业机会窗口（从 worker_guidance【增】信号延伸）
  - 合规红线、个体户 vs 小团队、从打工人→个体户的过渡节奏
  - 失败案例与诚实劝退（不是所有行业、所有人都适合创业）
- **对应动作链**：借势创业 + 开辟新业
- **方法论根基**：《丢掉幻想，准备斗争》（155）丢掉「等靠要」；《星星之火，可以燎原》（006）小火种起步；《关心群众生活》（011）从真实需求出发

### 层 2 · collaboration（协作与组织层）

- **核心问题**：单干做不出新蛋糕。普通劳动者怎么组织起来——合作社、互助小组、技能互换、跨行业协同、县域集体经济——把分散的劳动力拧成可持续的协作体。
- **为什么必要**：动作链里「结成协作」是唯一完全空白的动词；而且毛泽东方法论里的「统一战线 / 团结大多数 / 集中优势兵力」本来就有**集体版**，目前 worker_guidance 只用了个人版，是方法论资源的闲置。
- **与现有层的关系**：是 entrepreneurship 的天然配套——前者教「怎么单点突破」，本层教「怎么把单点连成网」。也与 worker_guidance 的转型路线图衔接（很多转型需要集体托底）。
- **初步内容结构**：
  - 协作形态谱系：合作社 / 互助组 / 技能互换 / 接包小队 / 县域集体经济
  - 信任与分配机制（怎么避免「合伙必散」的常见死法）
  - 跨行业协同案例（如「制造 + 内容 + 电商」三人组）
  - 合作的合规与红线（非法集资、传销边界、用工合规）
- **对应动作链**：结成协作
- **方法论根基**：《团结起来，划清敌我界限》（181）团结大多数；《关于正确处理人民内部矛盾的问题》（217）处理内部张力；《集中优势兵力，各个歼灭敌人》（103）集体版的聚焦

### 层 3 · growth_path（个人成长地图层）

- **核心问题**：worker_guidance 是「行业切片」（同行业不同人共用），缺一张「个人切片」（同一个人在不同阶段怎么走）。系统化「顺势成长」。
- **为什么必要**：现状的成长建议零散在各集群，读者要自己拼。一张按「起点画像 × 目标画像」组织的成长地图，能让一线劳动者直接对号入座。
- **与现有层的关系**：是 worker_guidance 的「个人视角」补充，不取代它。引用 worker_guidance 的技能建议，按人生阶段重新组织。
- **初步内容结构**：
  - 起点画像：应届生 / 30 岁产线工人 / 45 岁转行者 / 县域劳动者 / 退役/返乡人员
  - 每条线的技能学习地图、证书路径、免费/低成本学习资源
  - 关键节点决策（何时考证、何时转管理、何时独立）
- **对应动作链**：顺势成长
- **方法论根基**：《改造我们的学习》（060）面向真实需求；《论持久战》（027）成长是持久战

### 层 4 · new_value（新蛋糕机会地图层）

- **核心问题**：把「做出新蛋糕」（非零和创造）从口号变成可查的机会清单。跨集群提炼真增量，强调新需求/新业态而非存量内卷。
- **为什么必要**：是 entrepreneurship 和 worker_guidance 的**公共索引**——回答「机会在哪」，给前两者提供输入。analysis 已有「十大确定性方向」但偏宏观，本层下沉到可动手的颗粒度。
- **与现有层的关系**：横切层。向上引用 analysis 的确定性方向，向下给 entrepreneurship 和 worker_guidance 的【增】信号提供更细的「新业态」颗粒。
- **初步内容结构**：
  - 跨集群真增量清单：银发 / 低空 / 宠物 / 即时零售 / 技能服务化 / 银发文旅 / 县域消费…
  - 每个机会的「新」在哪（新需求 vs 新供给 vs 新组织方式）
  - 入场门槛、窗口期、饱和预警
- **对应动作链**：做出新蛋糕（+ 为开辟新业提供输入）
- **方法论根基**：《矛盾论》（017）抓增量这个主要矛盾；《星星之火，可以燎原》（006）识别真火种

### 层 5 · regional（地域 × 产业层，可选）

- **核心问题**：同一行业在长三角、中西部、县域的机会天差地别。analysis 提了区域战略篇，但没下沉到「某个地方的劳动者怎么办」。
- **为什么是可选**：地域颗粒度极细，工作量大，且时效性强（地方政策变动快）。建议在前四层稳定后再做，或作为前四层的横向标注而非独立大层。
- **与现有层的关系**：给所有行业层/创业层加一个「地域」滤镜。
- **初步内容结构**：
  - 三大动力源（长三角/珠三角/京津冀）vs 战略腹地 vs 县域 的机会差异
  - 重点行业的地方版机会（如低空在某试点城市、养老在某人口大省）
- **对应动作链**：为顺势成长 / 借势创业提供地域校准
- **方法论根基**：《矛盾论》（017）特殊性原理；《关心群众生活》（011）因地制宜

## 五、层间关系（文字版关系图）

```
                    references（原料：规划全文）
                          │
                          ▼
                      analysis（国家层：国家往哪走）
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
       new_value      worker_guidance  (十大确定性方向/新蛋糕机会)
      (机会在哪)      (行业怎么办)        │
            │             │             │
            └──────┬──────┘             │
                   ▼                    │
           entrepreneurship             │
           (怎么零成本启动) ◄────────────┘
                   │
                   ▼
           collaboration
           (怎么组织起来)
                   ▲
                   │（个人成长衔接）
           growth_path
           (个人怎么成长)
```

一句话理解：
- **analysis + worker_guidance** = 看清大势、站稳脚跟（已有，偏「择业/转型」）
- **new_value** = 机会在哪（横切索引，给行动层喂料）
- **entrepreneurship + collaboration** = 怎么创造、怎么协作（动作链后半段，偏「创业/组织」）
- **growth_path** = 个人怎么一步步成长（贯穿全程的个人视角）
- **regional** = 给以上所有层加地域滤镜（可选横切）

## 六、实施优先级与顺序建议

| 顺序 | 层 | 理由 |
|---|---|---|
| 1 | entrepreneurship | 直接补灵魂词「零成本创业」空白；复用现有 16 集群信号，启动成本最低 |
| 2 | collaboration | 动作链唯一完全空白的动词；与 entrepreneurship 是「单点突破 vs 连点成网」的天然一对 |
| 3 | growth_path | 系统化零散的成长建议，读者直接对号入座，性价比高 |
| 4 | new_value | 作为公共索引，前几层稳定后做能更好提炼；也可与层 1 并行做一个轻量版 |
| 5 | regional（可选） | 颗粒最细、时效最强，建议最后或作为横向标注 |

说明：层 1（entrepreneurship）和层 2（collaboration）构成「创业 + 协作」的核心组合，建议作为第一批一起规划；如果资源有限，先做层 1。new_value（层 4）虽然是「机会在哪」的起点判断，但因为 analysis + worker_guidance 已覆盖了大部分机会信号，它更多是「重新组织」而非「从无到有」，所以不必最先。

## 七、结构与命名规范（沿用既有约定）

为保证与现有 `worker_guidance/`、`analysis/` 风格一致，每个新层遵循：

- 目录名：英文小写（如 `entrepreneurship/`、`collaboration/`）
- 开篇文件：`00-方法论框架.md`（说明本层要解决什么、方法论根基、内容结构、诚实边界）
- 总索引：`README.md`（本层文件清单 + 跨层速查表 + 阅读建议）
- 正文文件：中文命名，按主题编号
- 每层都带「诚实的边界」一节：说清楚本层是参考框架不是承诺/投资建议，面向一线劳动者，语言朴素
- 方法论映射：每层都要回挂到 `methodology/great_man_inspiration/` 的具体著作编号，不做无根判断

## 八、诚实的边界（本规划文档自身）

1. 本文档是**规划**，不是已完成的产出。各层正文经确认后另起文件，本文档不替代任何正文。
2. 五个新增层的优先级是**建议**，最终顺序以确认为准；其中 regional 标注为可选，可能最终以横向标注形式融入其他层而非独立成层。
3. 新增层不是「为了多建目录而建目录」。每一层都必须能回答一个现有三层回答不了的问题，否则不立项。
4. 所有新增层同样遵守仓库总边界：面向一线劳动者、面向零成本/低成本、不提供投资/投机建议、不做政策复读机、不灌心灵鸡汤。
5. 本规划与 `README.md` 第二阶段目标对齐；若第二阶段表述日后调整，本规划相应校准。
